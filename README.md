# Consultant Toolkit — draw.io Plugin

A comprehensive consulting, strategy, and design plugin for [draw.io](https://app.diagrams.net). **272 frameworks** across 19 groups, from McKinsey problem-solving to AI system design patterns.

Each framework renders as a pre-built, editable diagram with styled cards, section labels, and integrated documentation. Fill in your own content directly on the canvas.

---

## Quick Start

1. Open draw.io (desktop or [app.diagrams.net](https://app.diagrams.net))
2. Go to **Extras → Plugins → Add**
3. Select `consultant-toolkit.js`
4. Click **Apply**, then reload
5. Use the toolbar buttons or `Ctrl+Shift+F` to open the Framework Picker

## Features

- **272 frameworks** — the largest consulting framework library for draw.io
- **Fillable worksheets** — 94 registered frameworks with interactive, editable card layouts
- **Display models** — 178 org-design frameworks rendered as structured diagrams
- **Framework Picker** with tag search, scenario guides ("I want to..."), recently-used quick access, and framework-vs-framework comparisons
- **Guided sequences** — step-by-step walkthroughs for GTM strategy and startup validation
- **Story documentation** — every framework includes When / Why / How / Case Study tooltips
- **Warm design system** — WCAG AA contrast, modular typography (20→16→14→12→11→10→9px), 5-level shadow scale, warm color palette
- **Auto-grouping** — each framework's cells are grouped for easy move/resize
- **Zero dependencies** — single self-contained JS file, no network requests

## Architecture

### Framework Types

| Type | Count | Description |
|------|-------|-------------|
| **Registered** | 94 | `FrameworkCatalog.register()` — full fillable diagrams with styled cards, section labels, and documentation |
| **orgFramework** | 178 | `orgFramework()` — display-only structured models with auto-grouping, section labels, and diagnostic callouts |

### Design System

| Token | Value | Usage |
|-------|-------|-------|
| `COLOR_BG` | `#FAFAF9` | Warm white background |
| `COLOR_INK` | `#0A0A0A` | Warm black text |
| `TYPE_HERO` | `fontSize=20;fontStyle=1` | Root framework title |
| `TYPE_H1` | `fontSize=16;fontStyle=1` | Section containers |
| `TYPE_H2` | `fontSize=14;fontStyle=1` | Card titles |
| `TYPE_BODY` | `fontSize=12` | Card content |
| `TYPE_CAPTION` | `fontSize=11` | Edge labels |
| `FW_SHADOW_SM` | `dy=2, opacity=6` | Subtle elevation (tooltips) |
| `FW_SHADOW_1` | `dy=3, opacity=10` | Cards |
| `FW_SHADOW_2` | `dy=5, opacity=16` | Containers |
| `FW_SHADOW_3` | `dy=10, opacity=22` | Root titles |

### Metadata Database

`framework-metadata.json` — a JSON database containing:

- **190** framework metadata entries (When / Why / How / Case Study)
- **20** group definitions (purpose, use case, effort)
- **95** framework-vs-framework comparison entries
- Injected into the plugin at build time via `FW_DB_START`/`FW_DB_END` markers

---

## Building

```bash
# Install (no dependencies needed — Node.js only)
# Build: injects framework-metadata.json into the plugin
node build.js
```

The build reads `framework-metadata.json` and injects it between the `FW_DB_START`/`FW_DB_END` markers in `consultant-toolkit.js`.

---

## Testing

```bash
node tests/harness.js
```

Runs all 322 framework execution tests + 29 action smoke tests. Current status: **322 PASS / 0 FAIL**.
